自适应Cell的两种方式

UI
  • tableview cell

iOS8之前

在iOS8之前,如果想在UITableView中展示可变高度的动态内容时候,我们需要手动的计算会发生变化控件的高度,如果你有多个动态的变化高度的控件, 那么你就需要计算多次…再统一相加起来,返回给cell的高度.具体实现实例参考我以前的博文UITableView详解(三)

iOS8之后

在iOS8中，苹果引入了UITableView的一项新功能–Self Sizing Cells,它的出现,能够让我们通过autolayout,很方便的展示动态控件的内容.主要代码: 

self.mainTableView.estimatedRowHeight = 20;

self.mainTableView.rowHeight = UITableViewAutomaticDimension;

PS:当然,前提是你已经为会变化高度的控件进行了约束,如图:

就是这么简单,相比8之前的方法,简直是减少了N堆代码
例子->

- (UITableViewCell *)tableView:(UITableView *)tableView cellForRowAtIndexPath:(NSIndexPath *)indexPath
{
    HMTTestTableViewCell *cell = [tableView dequeueReusableCellWithIdentifier:@"SizeCell" forIndexPath:indexPath];
    cell.nameLabel.text = @"HMT";
    cell.nameLabel.font = [UIFont preferredFontForTextStyle:UIFontTextStyleHeadline];
    cell.addressLabel.text = @"50万能买什么？也许是80多个IPHONE6，也许是无数大餐。但为了帮助贫困孩子，上海市世界外国语中学的魏琮泰、杨行同学和上海市世界外国语小学的魏启泰同学却毫不犹豫地捐出了积攒10余年的50万压岁钱，用于设立市慈善基金会“青春之光爱心专项基金”，而现年14岁的魏琮泰也成为沪上年龄最小的慈善专项基金主任。三名学生";
    cell.addressLabel.font = [UIFont preferredFontForTextStyle:UIFontTextStyleBody];
    return cell;
}

Warning

据说iOS8下的cell略坑,下面收集了2个问题,大家可以参考一下,可能就是你最近遇到的噢!
1.self size push之后的bug PS:貌似iOS8已经修复了这个问题
2.heightForRowAtIndexPath 在iOS8下重复计算问题

强烈推荐

尤其是第一篇和第二篇,两者结合可以解决UITableView能优化的极致,不过CoreText绘制LZ表示:smile:

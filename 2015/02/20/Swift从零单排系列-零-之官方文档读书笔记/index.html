Swift 官方文档读书笔记

前言

读完《The Swift Programming Language》后,对于Swift有了个大致的了解,但是还是缺少实际操作,以至于很多特性和注意点都不太熟悉,遂一一拿出做个总结,加上Swift语法毕竟还不稳定,实时更新,实时掌握!
注:基本概念和详细点请通读官方文档,第零篇以本人容易忽略的点为主,不是基础教学

###基本类型

  • Swift不支持布尔类型和非布尔类型的值之间的隐士类型转换(p75-p76)
  • 可选类型Optionals,值有或者无,格式:x?.无nil在Swift中并不是代表一个指针指向了一个不存在的对象而是代表任意类型的值的缺失情况.(p79)
  • 用if语句来判断一个可选是否有值(p80-p83)
    　１.确定有值,用可选类型后面+!的方式强制解封使用可选类型的值
    　２.可选绑定方式,判断并把值赋给一个常量或者变量.有值直接赋值并使用,没值if语句判断为假,不会赋值.
  • Array和Dictionary(p124-p141)
  • 待续……

###控制流

  • for-in,其中a...b表示a到b并且包括b;a..<b表示a到b但不包括b
  • _ 能够忽略具体的值,并且不提供循环遍历时对值的访问,表示我只需要for循环能够帮我执行正确的循环次数即可(p143-p145)
  • swift中的switch 不存在隐式的贯穿,每一个case分支都必须包含至少一条语句,这点不像C语言中可以同时匹配好几个条件.但如果确实需要,可以用关键字fallthrough实现.(p159)
  • switch中一个case也可以包含多个模式,用逗号分开
  • 待续……

###函数

  • 可变参数可以用来传入不确定数量的输入参数,格式是在变量类型后面加入...(p189)
    　１. 一个函数至多只能有一个可变参数
    　２.可变参数必须要在参数列表的最后一个
  • inout标记输入输出参数,只针对于变量(p192)
  • 待续……

###类和结构体

  • 延迟属性,用关键字lazy修饰,指第一次调用时才会计算其初始值的属性.使用情况 (p245):
    　１.如果类实例构造结束前无法知道具体值的外部信息
    　２.如果属性的值需要复杂或者大量计算时
    　３.必须声明为var

  • 计算属性,必须声明为var

    willSet{

    //newValue为参数

    }
    didSet{

    //oldValue为参数

    }
    ```

  • 下标方法,subcript(参数) ->返回值(get/set访问器)(p275)
  • override标记继承,final关键字来拒绝子类重写
  • 类属于引用类型,枚举和结构体属于值类型
  • 值类型的专属方法—Mutating(p267)
  • 可选用链,串联要访问的属性且在每个可选属性后面+?(p362)
  • 解决强引用的三种方法的使用场景(p334)
    　1.如果被指向的实例可能为nil,则用弱引用weak var 属性
    　2.如果被指向的实例不为nil,则用无主引用unowned var 属性
    　3.如果属性在使用闭包时,且闭包体内引用当前实例self而产生强引用环,使用捕获列表
    • is实现类型检查,左侧只能是一个父类的实例,右侧只能是一个子类实例
    • as实现了类型转换,又分as(可选格式)和as?(强制格式)->(p375)
    • extension(p390)不能扩展实例的存储属性,但是可以增加类型存储属性,带上关键字staic或class
    • 类类型只能增加便捷构造器,记得convenience关键字;而值类型,扩展构造器必须保证所有的属性都被初始化.
    • 类特定协议格式:@objc protocol xxx和protocox xxx:class
    • 协议遵循类在实现构造器需求时,构造器的定义必须使用required修饰,被final修饰的类除外
    • protocol本身也是类型,可以被存储,比如Array形式[xxxDelegate],里面存放的是该协议的遵循类型
    • 可选协议的声明必须用@objc修饰
    • 待续……

title: UIScrollView汇总(持续更新)
date: 2014-12-12 15:47:19
categories: UI

tags: UIScrollView

UIScrollView是iOS开发中可谓是最常用的UI控件(它的两个子类,不多说了…)了,介于经常被它一些莫名的问题所困扰,遂做个总结!(盗图,莫骂我😢)

基本用法

可直接参考我CSDN的博文(我就不搬过来了)

我遇到的问题

  • 某种情况下,在UIScrollView上添加子控件,尤其是VC,UIScrollView的contentSize被归于0
  • 最近一个项目,UIScrollView上放了5个VC(类似网易新闻),VC上放置2个UILabel,再用autolayout进行约束,结果->在iOS7系统~iPhone5 iPhone6上没问题,在iOS8系统~iPhone5上没问题,却在iOS8系统~iPhone6上出现了约束错误….随后不加scrollview进行验证,没有任何问题…
    @解决@: 
    -(void)viewDidLayoutSubviews {
        [super viewDidLayoutSubviews];
        // 重新设置UIScrollView的代码  
}
    viewDidLayoutSubviews的作用:view已经布局其Subviews。比如view的bounds改变了(例如状态栏从不显示到显示，视图方向变化)，已经调整Subviews的位置，在调整完成之后要做的一些工作就可以在该方法中实现.
